# Evaluation

TwinAgent AI evaluates anomaly detection against synthetic ground-truth fault labels generated by the simulator.

## Why evaluation is possible

The simulator injects known fault windows such as bearing wear, overheating, and belt misalignment. Because the generated dataset contains a `fault_label` column, we can compare detector outputs against known fault periods.

## Current metrics

The evaluation module computes:

- Precision
- Recall
- F1 score
- Accuracy
- False positive rate
- Detection delay
- Incident overlap score
- Incident-level diagnosis accuracy
- Row-level suspected-fault alignment

## Diagnosis metrics

TwinAgent AI separates two diagnosis metrics:

1. **Incident-level diagnosis accuracy**  
   This evaluates whether an incident's suspected fault matches any ground-truth fault active during that incident window. This is the most relevant diagnosis metric for the current MVP because the copilot explains incidents.

2. **Row-level suspected-fault alignment**  
   This evaluates each detected row inside fault windows. It is stricter and can be lower during overlapping or evolving fault windows.

## How to run

From the project root:

```cmd
python scripts\generate_synthetic_data.py
python scripts\run_anomaly_detection.py
python scripts\evaluate_detectors.py
```

The evaluation script writes:

```text
data/reports/evaluation_metrics.json
data/reports/evaluation_report.md
```

## Technical honesty

These metrics are evaluated on synthetic data. They are useful for comparing detector logic inside the prototype, but they must not be presented as real-world predictive-maintenance performance.
